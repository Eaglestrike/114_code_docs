  1. Work in branches NOT in main

We keep the latest working version of code in the main branch of the repository. If someone is working on a subsystem or an improvement, they create a separate branch and work there. This way we always have an easily accessible branch of working code. In general we try to keep the main branch clean.

  1. Create pull requests for code reviews OR to merge into main if

    your branch is tested and approved by the code lead
